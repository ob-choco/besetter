API 설계

GET /places/nearby
Query:
  latitude:  number (required)
  longitude: number (required)
  radius:    number (default: 100, meters)

Response: 200
[{
  id:        string
  name:      string
  type:      "gym" | "private-gym"
  latitude:  number
  longitude: number
  distance:  number (meters)
  createdBy: string (userId)
}]

// 필터 로직 (서버):
// type=gym → 모든 유저에게 반환
// type=private-gym → createdBy=me 일 때만 반환
POST /places
Body:
  name:      string (required)
  latitude:  number (required)
  longitude: number (required)
  type:      "gym" | "private-gym"
             (default: "gym")

Response: 201
{
  id:        string
  name:      string
  type:      "gym" | "private-gym"
  latitude:  number
  longitude: number
  createdBy: string (userId, 서버에서 설정)
}

Place 데이터 모델

collection: places
{
  _id:        ObjectId
  name:       String          // "더클라임 강남점"
  type:       String          // "gym" | "private-gym"
  latitude:   Number          // 37.516523
  longitude:  Number          // 127.019823
  createdBy:  ObjectId (ref)  // userId - 최초 등록자
  createdAt:  Date
}

// 인덱스: { latitude: 1, longitude: 1 }
// 또는 MongoDB 2dsphere index for geo queries

노출 규칙

type: "gym"
• 모든 유저의 nearby 검색에 노출
• 누구나 선택 가능
• 커뮤니티 공유 자산
type: "private-gym"
• createdBy 본인에게만 노출
• 다른 유저 검색 결과에 미포함
• 개인 연습 공간용
